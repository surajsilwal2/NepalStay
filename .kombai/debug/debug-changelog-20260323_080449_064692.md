## Debug Session — 20260323_080449_064692
**Issue:** "token invalid" error when paying through Khalti

## Root Cause
Both `app/api/payment/khalti/route.ts` and `app/api/payment/khalti/verify/route.ts` detect the Khalti environment using:

```typescript
const isTest = process.env.KHALTI_SECRET_KEY.startsWith("test_");
const khaltiBase = isTest ? "https://a.khalti.com" : "https://khalti.com";
```

The key `790beaf10e684cfd8f3e2658695e67b4` (missing the `test_secret_key_` prefix) caused `isTest = false`, routing all requests to `https://khalti.com` (live production). The live Khalti endpoint rejected the test/sandbox key with "token invalid".

## Changes Made

### 1. Fixed KHALTI_SECRET_KEY prefix in .env
- **File:** `.env`
- **Change:** Updated `KHALTI_SECRET_KEY` value from `"790beaf10e684cfd8f3e2658695e67b4"` to `"test_secret_key_790beaf10e684cfd8f3e2658695e67b4"`
- **Why:** Khalti test keys follow the format `test_secret_key_<hex>`. Adding the correct prefix makes `startsWith("test_")` return `true`, routing requests to `https://a.khalti.com` (the sandbox), and also sends the properly formatted key in the `Authorization: Key ...` header.
- **Revert:** Change back to `KHALTI_SECRET_KEY="790beaf10e684cfd8f3e2658695e67b4"`
