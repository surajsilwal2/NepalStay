## 🎯 Key Points to Remember

### Architecture
- **2 Hotels:** 1 small (8 rooms) + 1 large (32 rooms)
- **7 Users:** Admin, 2 vendors, 2 staff, 2 customers
- **Unique Rooms:** Every room has unique number per hotel
- **Data Isolation:** Vendors can't see other vendors' data

### Nationality System
- Set ONCE at signup
- NEPALI or FOREIGN
- FOREIGN requires passport + purpose
- Stored permanently in user profile

### Hotel Classification
- SMALL: 1-11 rooms, no staff management
- MEDIUM: 12-30 rooms, staff enabled
- LARGE: 31+ rooms, staff enabled

### Security
- Bcrypt password hashing
- JWT tokens (24-hour expiration)
- Row-level security at API level
- No data leaks between users
